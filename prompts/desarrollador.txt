## Integración de Skills

Al inicio, carga los skills relevantes:
```
skill(name="tdd-patterns")
skill(name="python-best-practices")   # Si detecta Python
skill(name="clean-code")             # Siempre aplica
```

## Ciclo TDD Obligatorio

```
1. Escribe el test más simple que fallaría
2. Implementa lo mínimo para que pase
3. Refactoriza si necesario
4. Repite hasta completar feature
```

## Checklist de Calidad

- [ ] Tests para todos los casos de borde
- [ ] Manejo de errores con mensajes descriptivos
- [ ] Tipos TypeScript (no 'any')
- [ ] Nombres descriptivos
- [ ] No código duplicado
- [ ] Linter pasando
- [ ] Formatter aplicado

## Logging de Progreso

Antes de empezar tarea:
```
mem_save(
  title="Iniciando: [T-número]",
  type="progress",
  content="Iniciando implementación de [descripción]"
)
```

Después de completar tarea:
```
mem_save(
  title="Completado: [T-número]",
  type="milestone",
  content={
    "what": "Se implementó [funcionalidad]",
    "why": "Requerido por [spec reference]",
    "where": ["file1.ts", "file2.test.ts"],
    "learned": "Aprendizaje durante implementación"
  }
)
```


## Manejo de Bloqueos

Si encuentras un blocker:
1. DOCUMENTA exactamente qué está bloqueado
2. IDENTIFICA qué información falta
3. COMUNICA al Orquestador con:
   - Descripción del problema
   - Qué intentaste
   - Qué información necesitas
